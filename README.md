# Charles Gonzalez – Personal Portfolio Website

## Project Overview
This project is a personal portfolio website created from scratch to professionally showcase my skills, background, and recent work. The website focuses on presenting my expertise in backend development, particularly in distributed systems, while providing a visually appealing, accessible, and interactive experience for potential employers.

---

## Target Audience
The primary audience for this website is **employers** in the software development industry, specifically those seeking backend developers with experience in distributed systems. The goal is to communicate my skills, experience, and career goals effectively to this audience.

---

## Content Strategy
The website is designed to clearly present the following information:

- **Professional Identity**: Full name, career goal, and short bio.
- **Skills Highlighted**: Backend development, distributed systems, and relevant technologies.
- **Projects Showcase**: Display of my most recent projects using interactive cards.
- **Contact Information**: Links to GitHub, LinkedIn, and a contact form for inquiries.

---

## Information Organization
The website includes the following pages/sections:

1. **Home** – Hero section introducing me and my skills.  
2. **Projects** – Interactive project cards showcasing my most recent work.  
3. **Skills** – Highlight of technologies and expertise.  
4. **About Me** – Professional bio and career goals.  
5. **Contact** – Form to initiate email and links to professional profiles.  
6. **Footer** – Quick links to resume, GitHub, LinkedIn, and copyright.

---

## Visual Design
- **Overall Style**: Minimalist, professional, and modern.  
- **Color Scheme**: Pinks, purples, and greys for a cohesive, attractive aesthetic ([Coolors palette used](https://coolors.co/)).  
- **Fonts**: Headings and body text use `"Roboto Slab", serif` for a clean and readable appearance.  
- **Layout**:
  - Navigation bar at the top of every page.
  - Hero section on homepage with brief intro and call-to-action buttons.
  - Project section in cards displayed in rows; clickable to open modal cards for details.
- **Responsiveness**: Media queries ensure compatibility across mobile and desktop devices.
- **Accessibility**: 
  - High-contrast text for readability.
  - Alt text for images.
  - Consistent font sizes.
- **Icons & Images**: 
  - Social icons for GitHub and LinkedIn.
  - Images for projects and personal profile.

## Interaction / Functionality
- **Navigation**: Smooth scrolling through sections via the navbar.  
- **Project Cards**: Clickable cards open modal windows to display project details.  
- **Contact Form**: Opens the default email client with a prefilled message for convenience.  
- **Hover Effects & Feedback**: Buttons, links, and cards change color on hover to provide interaction feedback.  
- **JavaScript Features**: Modal pop-ups, email link integration, and interactive buttons to enhance user experience.  

---

## Timeline / Project Milestones
1. **Planning & Content Development** – Identify audience, goals, and projects.  
2. **Wireframing & Design** – Sketch layouts and select color palette, fonts, and styles.  
3. **HTML & CSS Implementation** – Structure website and apply visual design.  
4. **JavaScript & Interactivity** – Add modal cards, hover effects, and contact functionality.  
5. **Testing & Responsiveness** – Ensure accessibility, cross-browser compatibility, and mobile friendliness.  
6. **Deployment** – Host on GitHub Pages and verify functionality.
